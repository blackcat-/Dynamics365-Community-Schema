Welcome the Dynamic 365 Schema Project

Dynamics 365 is coming and it is an exciting time that will allow businesses to implement powerful and adaptable applications with minimal development costs and a huge array of tools at their disposable. It will allow for a more natural interoperability between Microsoft Dynamics CRM, AX, Project Maderia, Office 365, Power Apps and Flow. To find out more about Dynamics 365, check the video out here: https://www.youtube.com/watch?v=fYIPXx9zjj8

The challenge is that within implementations, depending on the industry, there are often different customisations required as they store and use different data. This can be time consuming for an implementation and also challenging to create. However with the introduction of Dynamics 365 and the Common Data Model that it will use, there cannot be a better time to look at addressing this challenge.

The aim of the Dynamic 365 Community Schema project is initially utilising the Dynamics CRM out of the box schema for the Account and Contact entities, we all can work together on an open source schema for these record types that would then ultimately feed into the Common Data Model in Dynamics 365. The Schema's are intended to focus on data that is critical to the operation of the types of records across the world, and not just for local environments, so it is important that multiple different languages and industries are taken into consideration.

When an organisation takes on a Microsoft Dynamics CRM Project, they often customise these elements and that takes thought, effort and time. This project aims to reduce that effort so that organisations can simply import this Schema and not require the work associated to defining these record types. The intention is for it to grow well beyond the Account and Contact records.

How will this work?

The plan is that the solutions within CRM are broken into 'features' where then an organisation can adopt the schema's they need which will then combine to create their overall solution.

We need you! We need you to branch the main standard out of the box schema and put forward your ideas on what should be feature solutions to the overall Schema for the Account and Contact entities in Microsoft Dynamics CRM.

What can you do to get started?

The easiest thing you can do is begin raising issues 'under the 'Issues' section on the repository related to common issues we can face as a community. You can also begin to answer issues raised by other people.

Take a standard out of the box environment and create a new solution containing the entity you wish to customise. Make your edits to the fields and then once you have done this your ready to extract the XML using the custom built Schema Extractor available under this repository under the 'Schema Extractor' Folder.

If you make any edits that are in response to an issue raised by another member of the community, you can link that issue to your commit.

It's as simple as that to contribute to this project!

Using the Schema Generator

Scott (@scottdurow) has written a custom Schema Extractor tool that automatically exports the XML of your entities to a folder so you can easily commit your customisations directly to the project.

There is a 'How to' file on how to use the command line tool within the repository under the 'Schema Extractor' Folder.

...and after that?
